Template engine

Django's template engine is a built-in subsystem that constructs text output based on templates and context. A template is a text document, or a Python string, marked-up using the Django template language. A context is a dictionary mapping keys to their corresponding variable values. The template engine replaces variables with their values and executes tags which have special meanings.

Blocks

In Django templates, block tags define areas that child templates can override. In this exercise, title, stylesheet, and content are blocks that can be filled with content in templates that extend this one. This is useful for creating a base template that contains all the common elements of your site and defines blocks that child templates can override.

Loops

Django's template language includes a for tag for looping over items in a list. Inside the loop, you can use certain tag to display the current item.

if control structure

Django's template language includes if for conditional statements. You can use elif and else as well. The conditions in these statements can use variables from the context.

Display of Context Variables

You can display context variables in a Django template by wrapping the variable name in double curly braces. When the template engine renders the template, it replaces variable_name with the value of variable_name from the context. If variable_name is a complex object, you can access its attributes using dot notation, like variable_name.attribute.
